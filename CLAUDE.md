## 1. TL;DR
- **Что это:** браузерный arena-roguelike в духе Vampire Survivors. Тема: человек-**охотник на нечисть**
  (магия + оружие на стороне Света); главный/финальный босс — **вампир**. Язык интерфейса — русский.
- **Где код:** ОДИН файл `index.html` (~3550 строк, HTML+CSS+JS, без внешних библиотек). Canvas 2D + Web Audio API.
  Музыка — 5 собственных треков владельца в `music/` (`track1..5.mp3`), фон меню `art/menubg.jpg`, иконки PWA — в `icons/`, плюс `manifest.webmanifest` и `sw.js`.
- **Текущая версия:** 0.59 (константа `VERSION` в index.html). Каждое изменение = бамп версии (договорённость). АКТУАЛЬНЫЙ статус всех заходов/правок — в `ROADMAP.md` → «🗺️ КАРТА ЗАХОДОВ» (это живой леджер; данный файл — сжатая шпаргалка).
- **⚠️ ДВА агента правят `index.html` ПАРАЛЛЕЛЬНО:** геймплей-агент (этот — логика: оружие/враги/XP/экономика/боссы/музыка/`ETYPES`/поведения) и UI-агент (визуал: арт врагов `ENEMY_MAPS`, спрайты героев `HEROSPR`, меню/экраны/CSS/кнопки). Зоны НЕ пересекать. Мерж: UI-агент вливает `main` в свою ветку → `main` ff. См. память `longnight-agent-zones`.

## 3. Рабочий процесс (КАК тестировать и деплоить) — критично
**Локальный тест:**
- **ОСНОВНОЙ способ — Claude Preview MCP** (`mcp__Claude_Preview__preview_*`): `preview_start` (конфиг `.claude/launch.json`,
  имя `longnight`, `python3 -m http.server 8773`) → `preview_eval` (выполнить JS/проверки), `preview_screenshot`,
  `preview_console_logs` (level=error), `preview_click`. Навигация: `preview_eval` → `window.location.href='http://localhost:8773/index.html'`.
  ⚠️ Превью-вкладка в ФОНЕ троттлит rAF → `run.time` стоит; для проверки игровой механики вызывай функции вручную
  (напр. `bossUpdate` в цикле), а не жди тиков loop.
- Headless Chrome+CDP :9222 — ЗАПАСНОЙ вариант; ТОЛЬКО с `--mute-audio` и УБИВАТЬ после (`pkill -f "remote-debugging-port=9222"`),
  иначе процесс играет музыку «из ниоткуда» (был баг — владелец искал источник звука по компу).
- **Debug-API `window.__game`:** `start() spawn(type,n,opts) boss(i,fin) warp(sec) god(bool) kill() levelup()
  pick(i) clearEnemies() count() fps() gold(n) meta()`. Плюс все функции игры глобальны (refreshItems, damageEnemy,
  fireWeapons, bossUpdate, metaCost, …). **Дев-панель:** ОДИН тап по `#verCorner` (`toggleDev`), визуальная обёртка над `__game`
  (старт локации/героя, спавн любого босса, оружие/пассивки максом, warp/таймскейл, бог-мод, XP-лог `window.__xplog`).
- Синтаксис-чек без браузера: вырезать `<script>` и `node --check`:
  `sed -n '/<script>/,/<\/script>/p' index.html | sed '1d;$d' > /tmp/c.js && node --check /tmp/c.js`

**Деплой (GitHub Pages) — С 0.59 РЕПО = ИСТОЧНИК ИСТИНЫ, деплой = `git push`:**
- Локальная папка ПОДКЛЮЧЕНА к репо `emorozoff/long-night` (remote `origin`, ветка `main`). Игровой файл — `index.html`
  (переименован в 0.59; копировать ничего не надо). GitHub Pages служит `index.html` из `main` → **`git push origin main` = деплой.**
  Live: https://emorozoff.github.io/long-night/. Старый URL crimson-horde больше НЕ работает.
- **Два агента / ветки:** геймплей-агент работает на `main`, UI-агент — на своей ветке (вливает `main`→свою→`main` ff).
  Коммит — по команде владельца («коммить» = сохранить, «выложи» = push в `main` = на сайт).
- **GitHub-токен** лежит в macOS keychain: `printf "protocol=https\nhost=github.com\n" | git credential-osxkeychain get`
  → поле `password`. **gh CLI не установлен.** Аккаунт `emorozoff`.
- **CDN-лаг:** после пуша index.html обновляется ~1 мин, но новые БИНАРНИКИ (mp3/png) сначала отдают 404,
  затем 200 ещё через 1-2 мин — это нормально, подождать/перепроверить.
- **sw.js `CACHE`** (сейчас `'longnight-0.59'`) — ОБЯЗАТЕЛЬНО поднимать номер при каждом деплое, иначе у игроков
  залипают старые ассеты (документ грузится network-first, ассеты cache-first). ASSETS сейчас: `art/menubg.jpg`, иконки, `track1..5.mp3`.

## 4. Архитектура index.html
Один `<script>`, секции (искать по `// ===== НАЗВАНИЕ =====`, не по номерам строк — они плывут):
`UTILS → AUDIO → ЛОКАЦИИ → META → INPUT → TOUCH → SPRITES → RUN STATE → UPGRADES → FX HELPERS →
COMBAT → GEMS/XP → WEAPONS → ENEMY BEHAVIOUR → BOSS → DIRECTOR → PLAYER/BULLETS/GEMS → UPDATE →
RENDER → MUSIC → MAIN LOOP → UI/SCREENS → WIRING → DEBUG API → BOOT`.

**Глобальное состояние:** `state` (строка: menu/heroes/pickLoc/shop/stats/best/settings/playing/paused/levelup/dead),
`run` (объект забега, пересоздаётся в `newRun()`), `meta` (сейв, localStorage + санитизация).

**Ключевые системы и функции:**
- **Спрайты:** `mkCanvas`, `px(rows,pal,s,outline)` — пиксель-арт из карты символов; `addOutline`, `whiteOf`
  (белая версия для вспышки урона). Враги — таблица `ENEMY_MAPS` (2 кадра ходьбы через `stepRows`/`shiftRow`).
  `SPR.*` — все спрайты (враги `SPR[type]`/`_a`/`_w`/`_wa`, снаряды `SPR.blade/scythe/pbolt/pbolt_g`,
  пикапы, иконки `SPR.icons`, тайлы `SPR.grounds`, декор `SPR.decors`).
- **Аудио:** объект `A` — синтез SFX + (отключённая в 0.15) процедурная музыка. `A.tone/noise`, компрессор. `A.coin()` = звон монеты.
  Файловая музыка — объект `MUSIC` (5 треков владельца `track1..5.mp3`; роли menu=t1/calm=t3/intense=t2/boss=t4/tension+final=t5;
  **бесшовный кроссфейд-луп** через 2 `<audio>` на «деку» — `applyDeck`; меню прослушивания в настройках — `buildMusicTest`).
  Тишина при master=0 — `A.ctx.suspend()` + разрыв цепи. Громкость синкается между вкладками (storage-событие).
- **Бой:** `fireWeapons(dt)` (все 8 орудий), `fireChain`, `damageEnemy(e,dmg,src,kx,ky)`, `damagePlayer(d,src)`,
  `killEnemy(e)` (дроп/комбо/делёж/осколки), `addBurst/addText` (партиклы/цифры), `hitStopAdd`, `flashGold`.
- **Враги:** `spawnEnemy(type,x,y,opts)`; поведение — switch по `e.behav` в `updateEnemies(dt)` (chase/swarm/hop/ranged/tank/warp/fast/exploder; паук — split в `killEnemy`). Босс — `bossUpdate`→`BOSS_MECH[bkind]` (механика у ВСЕХ 8: matron/maneater/flayer/drowned/shepherd/gravedigger/maiden/deadking), `spawnBoss(idx,isFinal)`, ростер `BOSSES` (8). Боссовые зоны: `run.flayPools/tides/tideZones/torches`; неуязвимость короля — флаг `e.invuln` (гейт в damageEnemy); фантомы Девы — `e.phantom` (гейт в killEnemy). `WIN_TIME=900`=15:00 → ВЕЧНЫЙ ВАМПИР → `winGame()`.
- **Спавн/сложность:** `director(dt)`, `diffNow()`, `pickType(t)` (трикл-фоддер), `doWave(t)` (конечные тематические группы опасных типов), элита/боссы по таймерам.
- **Левел-ап:** `addXP` → `openChoices` → `rollChoices`(`rollUpgrades`+эволюции `EVOS`+слияния `FUSIONS`)/`renderChoices`/`renderLvlActions`.
  `cardClick`→`pickChoice`/`banishCard`; `doReroll`/`toggleBanish`/`doSkip`. `applyUpgrade` (kind w/p/evo/fusion). Слияния — `isFused()`, фиолетовая связь в HUD `.ichip.fused`.
- **Зоны:** `updateZones` (лужи swamp/crypt, разломы ash/inferno, кислотные лужи/горшки mortar, волны bell),
  `inPool/poolAt`. **Рендер:** `render()` → drawGround/drawZones/drawAura/drawGems/drawEnemies/drawPlayer/
  drawBlades/drawBullets/drawPots/drawFx/drawParts/drawTexts/drawMenuAmbient + торч-свет + виньетка + fps.
- **UI:** `setState(s)` + `SCREEN_MAP`, `refreshHud`, `refreshItems` (HUD: 4 слота оружия + пассивки),
  `buildHeroes/buildPickLoc/buildShop/buildStats/buildBestiary`, `icoURL/sprURL`, `recordRun`, `checkAchievements`.
- **Мета:** `defaultMeta/sanitizeMeta` (САНИТИЗАЦИЯ всех полей — не доверять localStorage), `metaBonus`, `saveMeta`.

## 7. Известные баги / подводные камни (hard-won)
- **Retina/DPR:** канвасу нужны И `cv.width=W*DPR`, И `cv.style.width=W+'px'` — иначе на Retina картинка
  уезжает в угол и мылится. Есть `watchDPR()` на смену dppx (перенос окна между мониторами).
- **TDZ:** `sanitizeMeta` НЕ должна ссылаться на `HEROES`/`EVOS` (объявлены позже) — вызывается при загрузке.
  Валидацию hero делать без списка (fallback в newRun/buildHeroes).
